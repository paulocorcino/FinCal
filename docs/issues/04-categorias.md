# Categorias: CRUD com Tipo

## What to build

CRUD de **Categoria**, cada uma com um **Tipo** associado (Receita ou Despesa). Sem hierarquia (não há subcategorias); cor/ícone são cosméticos. As Categorias padrão semeadas no registro (Slice 02) aparecem aqui e são editáveis. Tela com `EmptyState` e exclusão via `AlertDialog`.

## Acceptance criteria

- [ ] Criar/editar/excluir Categoria com **Tipo** (Receita/Despesa) e cor/ícone cosméticos
- [ ] Sem subcategorias (estrutura plana)
- [ ] Categorias semeadas no registro aparecem e são editáveis
- [ ] `EmptyState` padronizado; exclusão via `AlertDialog`; queries filtram por `userId`
- [ ] Evidência: teste do CRUD + screenshot
